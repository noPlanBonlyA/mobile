**Преимущества:**
- ✅ Интеграция с GitHub

## 3. Архитектура CI/CD Pipeline

### 3.1. Стадии Pipeline

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Checkout   │────▶│    Build &   │────▶│     Test     │────▶│    Deploy    │
│     Code     │     │     Lint     │     │              │     │  to Server   │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
       │                     │                     │                     │
       │                     │                     │                     │
       ▼                     ▼                     ▼                     ▼
   git clone          • Build Docker      • Unit tests        • SSH to server
   from GitHub        • Python lint       • API tests         • Pull images
                      • Type checking     • Postman tests     • docker-compose up
                      • Security scan                         • Health check
```

### 3.2. Триггеры

```yaml
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]
  release:
    types: [created]
```

## 4. Настройка удаленного сервера

### 4.1. Требования к серверу

**Минимальные требования:**
- OS: Ubuntu 20.04/22.04 LTS
- CPU: 2 cores
- RAM: 4 GB
- Disk: 20 GB SSD
- Docker 24.0+
- Docker Compose v2+

### 4.2. Подготовка сервера

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Добавление пользователя в группу docker
sudo usermod -aG docker $USER

# Установка Docker Compose
sudo apt install docker-compose-plugin

# Проверка
docker --version
docker compose version
```

### 4.3. Настройка SSH доступа

```bash
# На локальной машине: генерация SSH ключа
ssh-keygen -t ed25519 -C "github-actions-deploy" -f ~/.ssh/deploy_key

# Копирование публичного ключа на сервер
ssh-copy-id -i ~/.ssh/deploy_key.pub user@your-server.com

# Тест подключения
ssh -i ~/.ssh/deploy_key user@your-server.com
```

## 5. GitHub Actions Workflows

### 5.1. Main CI/CD Workflow

**Файл:** `.github/workflows/deploy.yml`

```yaml
name: CI/CD Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:
  DOCKER_REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  # Job 1: Lint and Test
  test:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.12'
      
      - name: Install dependencies
        run: |
          pip install uv
          cd Production\ 46/backendITSchool
          uv sync
      
      - name: Run linting
        run: |
          cd Production\ 46/backendITSchool
          uv run ruff check .
          uv run mypy school_site/
      
      - name: Run unit tests
        run: |
          cd Production\ 46/backendITSchool
          uv run pytest tests/ -v --cov=school_site
      
      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./Production\ 46/backendITSchool/coverage.xml

  # Job 2: Build Docker Images
  build:
    runs-on: ubuntu-latest
    needs: test
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    
    permissions:
      contents: read
      packages: write
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
      
      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3
      
      - name: Log in to GitHub Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.DOCKER_REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Extract metadata for Main Backend
        id: meta-backend
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.DOCKER_REGISTRY }}/${{ env.IMAGE_NAME }}/backend
          tags: |
            type=ref,event=branch
            type=sha,prefix={{branch}}-
            type=semver,pattern={{version}}
      
      - name: Build and push Main Backend
        uses: docker/build-push-action@v5
        with:
          context: ./Production\ 46
          file: ./Production\ 46/docker/backend/Dockerfile
          push: true
          tags: ${{ steps.meta-backend.outputs.tags }}
          labels: ${{ steps.meta-backend.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
      
      - name: Extract metadata for Email Service
        id: meta-email
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.DOCKER_REGISTRY }}/${{ env.IMAGE_NAME }}/email-service
      
      - name: Build and push Email Service
        uses: docker/build-push-action@v5
        with:
          context: ./Production\ 46
          file: ./Production\ 46/docker/email/Dockerfile
          push: true
          tags: ${{ steps.meta-email.outputs.tags }}
          labels: ${{ steps.meta-email.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # Job 3: Deploy to Server
  deploy:
    runs-on: ubuntu-latest
    needs: build
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    
    environment:
      name: production
      url: https://api.itschool.example.com
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
      
      - name: Setup SSH
        uses: webfactory/ssh-agent@v0.8.0
        with:
          ssh-private-key: ${{ secrets.SSH_PRIVATE_KEY }}
      
      - name: Add server to known hosts
        run: |
          ssh-keyscan -H ${{ secrets.SERVER_HOST }} >> ~/.ssh/known_hosts
      
      - name: Create deployment directory on server
        run: |
          ssh ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }} '
            mkdir -p ~/deployments/itschool
          '
      
      - name: Copy docker-compose to server
        run: |
          scp -r ./Production\ 46/docker-compose.yml \
            ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }}:~/deployments/itschool/
      
      - name: Copy environment file
        run: |
          echo "${{ secrets.ENV_FILE }}" > .env
          scp .env ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }}:~/deployments/itschool/
      
      - name: Deploy application
        run: |
          ssh ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }} '
            cd ~/deployments/itschool
            
            # Pull latest images
            docker compose pull
            
            # Stop old containers
            docker compose down
            
            # Start new containers
            docker compose up -d
            
            # Wait for services to be healthy
            sleep 30
            
            # Check services status
            docker compose ps
          '
      
      - name: Health check
        run: |
          attempt=0
          max_attempts=10
          
          until curl -f https://api.itschool.example.com/docs || [ $attempt -eq $max_attempts ]; do
            echo "Waiting for API to be ready... ($attempt/$max_attempts)"
            attempt=$((attempt+1))
            sleep 10
          done
          
          if [ $attempt -eq $max_attempts ]; then
            echo "API failed to start!"
            exit 1
          fi
          
          echo "API is healthy!"
      
      - name: Run post-deployment tests
        run: |
          # Можно запустить Newman тесты или другие smoke tests
          echo "Running smoke tests..."
      
      - name: Rollback on failure
        if: failure()
        run: |
          ssh ${{ secrets.SERVER_USER }}@${{ secrets.SERVER_HOST }} '
            cd ~/deployments/itschool
            docker compose down
            git checkout HEAD~1  # откат к предыдущему коммиту
            docker compose up -d
          '
      
      - name: Send deployment notification
        if: always()
        uses: 8398a7/action-slack@v3
        with:
          status: ${{ job.status }}
          text: 'Deployment to production: ${{ job.status }}'
          webhook_url: ${{ secrets.SLACK_WEBHOOK }}
```

### 5.2. Pull Request Workflow

**Файл:** `.github/workflows/pr-check.yml`

```yaml
name: PR Checks

on:
  pull_request:
    types: [opened, synchronize, reopened]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.12'
      
      - name: Install dependencies
        run: |
          pip install uv
          cd Production\ 46/backendITSchool
          uv sync
      
      - name: Lint with ruff
        run: |
          cd Production\ 46/backendITSchool
          uv run ruff check . --output-format=github
      
      - name: Type check with mypy
        run: |
          cd Production\ 46/backendITSchool
          uv run mypy school_site/ --show-error-codes
      
      - name: Run tests
        run: |
          cd Production\ 46/backendITSchool
          uv run pytest tests/ -v
      
      - name: Comment PR
        uses: actions/github-script@v7
        if: always()
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: '✅ All checks passed! Ready for review.'
            })
```

### 5.3. Scheduled Security Scan

**Файл:** `.github/workflows/security.yml`

```yaml
name: Security Scan

on:
  schedule:
    - cron: '0 0 * * 1'  # Каждый понедельник в 00:00
  workflow_dispatch:

jobs:
  security:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Run Trivy vulnerability scanner
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: 'fs'
          scan-ref: './Production 46'
          format: 'sarif'
          output: 'trivy-results.sarif'
      
      - name: Upload Trivy results to GitHub Security
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: 'trivy-results.sarif'
      
      - name: Scan for secrets
        uses: trufflesecurity/trufflehog@main
        with:
          path: ./
          base: main
          head: HEAD
```

## 7. Настройка сервера для непрерывного развертывания

### 7.1. Nginx как Reverse Proxy

**Файл на сервере:** `/etc/nginx/sites-available/itschool`

```nginx
server {
    listen 80;
    server_name api.itschool.example.com;
    
    # Redirect to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.itschool.example.com;
    
    # SSL certificates (Let's Encrypt)
    ssl_certificate /etc/letsencrypt/live/api.itschool.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.itschool.example.com/privkey.pem;
    
    # Security headers
    add_header Strict-Transport-Security "max-age=31536000" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    
    # Main Backend
    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # Swagger docs
    location /docs {
        proxy_pass http://localhost:8080/docs;
    }
    
    # MinIO
    location /files/ {
        proxy_pass http://localhost:9000/;
        proxy_set_header Host $host;
    }
}
```

### 7.3. Systemd Service для автоматического запуска

**Файл:** `/etc/systemd/system/itschool.service`

```ini
[Unit]
Description=IT School Application
Requires=docker.service
After=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/home/deploy/deployments/itschool
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
TimeoutStartSec=0

[Install]
WantedBy=multi-user.target
```

```bash
# Включение service
sudo systemctl enable itschool.service
sudo systemctl start itschool.service
```

## 9. Best Practices

### 9.1. Безопасность

✅ **Никогда не коммитить секреты**  
✅ Использовать GitHub Secrets  
✅ Ротация SSH ключей  
✅ Минимальные права доступа  
✅ Security scanning в CI

### 9.2. Надежность

✅ **Rollback стратегия** при ошибках  
✅ Health checks после деплоя  
✅ Blue-Green deployment для zero-downtime  
✅ Backup перед обновлением  
✅ Мониторинг после деплоя

### 9.3. Производительность

✅ **Docker layer caching**  
✅ Параллельные jobs  
✅ Инкрементальные тесты  
✅ Артефакты между jobs

## 10. Альтернатива: GitLab CI

**Файл:** `.gitlab-ci.yml`

```yaml
stages:
  - test
  - build
  - deploy

variables:
  DOCKER_DRIVER: overlay2
  DOCKER_TLS_CERTDIR: "/certs"

test:
  stage: test
  image: python:3.12-slim
  script:
    - pip install uv
    - cd "Production 46/backendITSchool"
    - uv sync
    - uv run pytest tests/

build:
  stage: build
  image: docker:24.0
  services:
    - docker:24.0-dind
  script:
    - docker build -t $CI_REGISTRY_IMAGE/backend:$CI_COMMIT_SHA -f docker/backend/Dockerfile .
    - docker push $CI_REGISTRY_IMAGE/backend:$CI_COMMIT_SHA
  only:
    - main

deploy:
  stage: deploy
  image: alpine:latest
  before_script:
    - apk add --no-cache openssh-client
    - eval $(ssh-agent -s)
    - echo "$SSH_PRIVATE_KEY" | tr -d '\r' | ssh-add -
  script:
    - ssh $SERVER_USER@$SERVER_HOST "
        cd ~/deployments/itschool &&
        docker compose pull &&
        docker compose up -d
      "
  only:
    - main
  environment:
    name: production
    url: https://api.itschool.example.com
```
