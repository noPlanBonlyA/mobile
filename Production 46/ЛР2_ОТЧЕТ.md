## 2. Принципы REST

### 2.1. Основные концепции

**REST (Representational State Transfer)** - архитектурный стиль для создания веб-сервисов, основанный на следующих принципах:

1. **Клиент-серверная архитектура** - разделение ответственности между клиентом и сервером
2. **Stateless** - каждый запрос содержит всю необходимую информацию
3. **Кэширование** - ответы могут кэшироваться
4. **Единообразие интерфейса** - стандартизированные HTTP методы и коды ответов
5. **Layered system** - слоистая архитектура
6. **Code on demand** (опционально) - возможность передачи исполняемого кода

### 2.2. HTTP методы

- **GET** - получение ресурса (идемпотентный, safe)
- **POST** - создание нового ресурса
- **PUT** - полное обновление ресурса (идемпотентный)
- **PATCH** - частичное обновление ресурса
- **DELETE** - удаление ресурса (идемпотентный)

### 2.3. HTTP коды ответов

- **200 OK** - успешный GET/PUT/PATCH
- **201 Created** - успешное создание (POST)
- **204 No Content** - успешное удаление (DELETE)
- **400 Bad Request** - ошибка валидации
- **401 Unauthorized** - требуется аутентификация
- **403 Forbidden** - доступ запрещен
- **404 Not Found** - ресурс не найден
- **409 Conflict** - конфликт (duplicate)
- **500 Internal Server Error** - ошибка сервера

## 4. Структура реализации

### 4.1. Роутер (Router Layer)

```python
# apps/courses/router.py
from fastapi import APIRouter, Depends, status
from uuid import UUID
from .schemas import CourseCreate, CourseResponse
from .services import CourseService
from .depends import get_course_service

router = APIRouter(prefix="/api/courses", tags=["courses"])

@router.post("/", response_model=CourseResponse, status_code=status.HTTP_201_CREATED)
async def create_course(
    data: CourseCreate,
    service: CourseService = Depends(get_course_service)
):
    """Создание нового курса"""
    return await service.create_course(data)

@router.get("/{course_id}", response_model=CourseResponse)
async def get_course(
    course_id: UUID,
    service: CourseService = Depends(get_course_service)
):
    """Получение курса по ID"""
    return await service.get_course(course_id)

@router.put("/{course_id}", response_model=CourseResponse)
async def update_course(
    course_id: UUID,
    data: CourseUpdate,
    service: CourseService = Depends(get_course_service)
):
    """Обновление курса"""
    return await service.update_course(course_id, data)

@router.delete("/{course_id}", status_code=status.HTTP_204_NO_CONTENT)
async def delete_course(
    course_id: UUID,
    service: CourseService = Depends(get_course_service)
):
    """Удаление курса"""
    await service.delete_course(course_id)
```

### 4.2. Сервисный слой (Service Layer)

```python
# apps/courses/services/course_service.py
from uuid import UUID
from typing import List
from ..repositories import CourseRepository
from ..schemas import CourseCreate, CourseUpdate
from ..models import Course

class CourseService:
    def __init__(self, repository: CourseRepository):
        self.repository = repository
    
    async def create_course(self, data: CourseCreate) -> Course:
        """Создание курса с валидацией"""
        # Проверка уникальности названия
        existing = await self.repository.get_by_name(data.name)
        if existing:
            raise ValueError("Course with this name already exists")
        
        return await self.repository.create(**data.model_dump())
    
    async def get_course(self, course_id: UUID) -> Course:
        """Получение курса с проверкой существования"""
        course = await self.repository.get_by_id(course_id)
        if not course:
            raise ValueError("Course not found")
        return course
    
    async def update_course(self, course_id: UUID, data: CourseUpdate) -> Course:
        """Обновление курса"""
        course = await self.get_course(course_id)
        return await self.repository.update(course_id, **data.model_dump(exclude_unset=True))
    
    async def delete_course(self, course_id: UUID) -> bool:
        """Удаление курса"""
        return await self.repository.delete(course_id)
```

### 4.3. Репозиторий (Repository Layer)

```python
# apps/courses/repositories/course_repository.py
from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession
from school_site.core.repositories import BaseRepository
from ..models import Course

class CourseRepository(BaseRepository[Course]):
    def __init__(self, session: AsyncSession):
        super().__init__(Course, session)
    
    async def get_by_name(self, name: str):
        result = await self.session.execute(
            select(Course).where(Course.name == name)
        )
        return result.scalar_one_or_none()
```

## 6. Соответствие принципам REST

### 6.1. Единообразие URI

✅ **Правильно:**
- `/api/users` - коллекция
- `/api/users/{id}` - конкретный ресурс
- `/api/courses/{id}/lessons` - вложенный ресурс

❌ **Неправильно:**
- `/api/getUser?id=123`
- `/api/user/delete/{id}`

### 6.2. HTTP методы

✅ **Правильно:**
- `GET /api/courses` - получение списка
- `POST /api/courses` - создание
- `PUT /api/courses/{id}` - обновление
- `DELETE /api/courses/{id}` - удаление

### 6.3. Stateless

Каждый запрос содержит всю необходимую информацию:
- Аутентификация через токены в cookies/headers
- Нет серверной сессии
- Контекст запроса в параметрах

### 6.4. HATEOAS (частично)

```json
{
  "id": "uuid",
  "name": "Python для начинающих",
  "_links": {
    "self": "/api/courses/uuid",
    "lessons": "/api/courses/uuid/lessons",

  }
```

```bash
    "description": "Курс по основам Python",
    "age_category": ["ADULT"],
    "price": 5000
  }'

# Получение курса
curl http://localhost:8080/api/courses/{course_id}

curl -X PUT http://localhost:8080/api/courses/{course_id} \
  -H "Content-Type: application/json" \
  -d '{

  }'
```
