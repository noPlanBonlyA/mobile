## 1. Выбор варианта проекта

Выбран вариант: **Платформа для онлайн-школы IT**

### Описание проекта
Платформа для управления образовательным процессом IT-школы с функционалом:
- Система управления пользователями (студенты, преподаватели, администраторы)
- Управление курсами и уроками
- Система групп и расписания занятий
- Отслеживание прогресса студентов
- Система баллов и достижений
- Новости и уведомления
- Магазин товаров для студентов

## 2. Проектирование базы данных

### 2.1. Основные сущности

#### Users (Пользователи)
- **id** (UUID, PK) - уникальный идентификатор
- **first_name** (String) - имя
- **surname** (String) - фамилия
- **patronymic** (String) - отчество
- **username** (String, unique) - логин пользователя
- **phone_number** (String) - номер телефона
- **email** (String, unique) - электронная почта
- **password_hash** (String) - хеш пароля
- **role** (Enum: STUDENT, TEACHER, ADMIN) - роль пользователя
- **birth_date** (Date) - дата рождения
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### Students (Студенты)
- **id** (UUID, PK) - уникальный идентификатор
- **user_id** (UUID, FK → Users, unique) - связь с пользователем
- **points** (Integer) - количество баллов студента
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### Teachers (Преподаватели)
- **id** (UUID, PK) - уникальный идентификатор
- **user_id** (UUID, FK → Users, unique) - связь с пользователем
- **bio** (Text) - биография преподавателя
- **specialization** (String) - специализация
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### Courses (Курсы)
- **id** (UUID, PK) - уникальный идентификатор
- **name** (String, unique) - название курса
- **description** (Text) - описание курса
- **age_category** (Array[Enum]) - возрастные категории
- **price** (Integer) - стоимость курса
- **author_name** (String) - автор курса
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### Lessons (Уроки)
- **id** (UUID, PK) - уникальный идентификатор
- **course_id** (UUID, FK → Courses) - связь с курсом
- **title** (String) - название урока
- **description** (Text) - описание урока
- **lesson_number** (Integer) - номер урока
- **level** (Integer) - уровень сложности
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### Groups (Группы)
- **id** (UUID, PK) - уникальный идентификатор
- **name** (String, unique) - название группы
- **description** (Text) - описание группы
- **teacher_id** (UUID, FK → Teachers) - преподаватель группы
- **start_date** (Date) - дата начала обучения
- **end_date** (Date) - дата окончания обучения
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### Events (События/Мероприятия)
- **id** (UUID, PK) - уникальный идентификатор
- **name** (String) - название события
- **description** (Text) - описание события
- **date** (DateTime) - дата и время проведения
- **location** (String) - место проведения
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### News (Новости)
- **id** (UUID, PK) - уникальный идентификатор
- **title** (String) - заголовок новости
- **content** (Text) - содержание новости
- **author_id** (UUID, FK → Users) - автор новости
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

#### Products (Товары)
- **id** (UUID, PK) - уникальный идентификатор
- **name** (String) - название товара
- **description** (Text) - описание товара
- **price** (Integer) - стоимость в баллах
- **quantity** (Integer) - количество на складе
- **created_at** (DateTime) - дата создания
- **updated_at** (DateTime) - дата обновления

### 2.2. Связующие таблицы (Many-to-Many)

#### GroupStudent (Студенты в группах)
- **id** (UUID, PK)
- **group_id** (UUID, FK → Groups)
- **student_id** (UUID, FK → Students)
- Constraint: unique(group_id, student_id)

#### CourseStudent (Студенты на курсах)
- **id** (UUID, PK)
- **course_id** (UUID, FK → Courses)
- **student_id** (UUID, FK → Students)
- **progress** (Float) - прогресс прохождения курса
- Constraint: unique(course_id, student_id)

#### LessonGroup (Уроки для групп)
- **id** (UUID, PK)
- **lesson_id** (UUID, FK → Lessons)
- **group_id** (UUID, FK → Groups)
- **scheduled_at** (DateTime) - запланированное время урока
- **is_completed** (Boolean) - статус проведения
- Constraint: unique(lesson_id, group_id)

#### LessonStudent (Прогресс студента по урокам)
- **id** (UUID, PK)
- **lesson_id** (UUID, FK → Lessons)
- **student_id** (UUID, FK → Students)
- **is_completed** (Boolean) - статус завершения урока
- **attended** (Boolean) - посещение урока
- **grade** (Float) - оценка за урок

#### EventsUsers (Участники событий)
- **id** (UUID, PK)
- **event_id** (UUID, FK → Events)
- **user_id** (UUID, FK → Users)
- Constraint: unique(event_id, user_id)

### 3.1. Основные связи:

1. **User → Student/Teacher** (1:1)
   - Один пользователь может быть либо студентом, либо преподавателем

2. **Student ↔ Group** (M:N через GroupStudent)
   - Студент может обучаться в нескольких группах
   - В группе может быть много студентов

3. **Student ↔ Course** (M:N через CourseStudent)
   - Студент может записаться на несколько курсов
   - На курс может записаться много студентов

4. **Course → Lesson** (1:M)
   - Курс содержит множество уроков
   - Урок принадлежит одному курсу

5. **Group ↔ Lesson** (M:N через LessonGroup)
   - Группа может проходить много уроков
   - Урок может быть назначен нескольким группам

6. **Student ↔ Lesson** (M:N через LessonStudent)
   - Отслеживание прогресса студента по каждому уроку

7. **Teacher → Group** (1:M)
   - Преподаватель ведет несколько групп
   - Группу ведет один преподаватель

8. **User ↔ Event** (M:N через EventsUsers)
   - Пользователь может участвовать в нескольких событиях
   - В событии участвует много пользователей

## 4. Ограничения и бизнес-правила

### 4.1. Проверки целостности (Constraints)

1. **Положительные значения:**
   - `students.points >= 0`
   - `courses.price >= 0 OR price IS NULL`
   - `products.price >= 0`
   - `products.quantity >= 0`

2. **Уникальность:**
   - `users.username` - уникальный логин
   - `users.email` - уникальная почта
   - `courses.name` - уникальное название курса
   - `groups.name` - уникальное название группы
   - `group_student(group_id, student_id)` - студент не может быть дважды в одной группе
   - `course_student(course_id, student_id)` - студент не может записаться дважды на курс

3. **Обязательные поля:**
   - Все пользователи должны иметь email, username, password_hash, role, birth_date
   - Все курсы должны иметь name, description, age_category
   - Все группы должны иметь name, description, start_date, end_date

### 4.2. Каскадное удаление (ON DELETE CASCADE)

1. **User → Student/Teacher** - при удалении пользователя удаляется связанный профиль студента/преподавателя
2. **Student → GroupStudent** - при удалении студента удаляются все связи с группами
3. **Student → LessonStudent** - при удалении студента удаляется весь прогресс по урокам
4. **Group → LessonGroup** - при удалении группы удаляются все связи с уроками
5. **User → RefreshToken** - при удалении пользователя удаляются все токены

### 4.3. Бизнес-логика

1. **Даты групп:**
   - `end_date` должна быть больше `start_date`

2. **Роли пользователей:**
   - STUDENT - может записываться на курсы, получать баллы, покупать товары
   - TEACHER - может вести группы, оценивать студентов
   - ADMIN - полный доступ к системе

3. **Баллы студентов:**
   - Начальное значение: 0
   - Могут начисляться за посещение уроков, выполнение заданий
   - Могут списываться при покупке товаров
   - История изменений хранится в PointsHistory

## 6. Нормализация

База данных спроектирована в соответствии с **3НФ (третьей нормальной формой)**:

1. **1НФ:** Все атрибуты атомарны, нет повторяющихся групп
2. **2НФ:** Все неключевые атрибуты полностью зависят от первичного ключа
3. **3НФ:** Отсутствуют транзитивные зависимости между неключевыми атрибутами
