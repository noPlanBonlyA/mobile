## 2. Документирование через Swagger/OpenAPI

### 2.1. Автоматическая документация FastAPI

FastAPI автоматически генерирует OpenAPI документацию. Настройка приложения:

**Файл:** [bootstrap.py](Production 46/backendITSchool/school_site/bootstrap.py)

```python
def create_app() -> FastAPI:
    app = FastAPI(
        title="IT School API",
        description="RESTful API для платформы онлайн IT-школы",
        version="1.0.0",
        docs_url='/docs',           # Swagger UI
        redoc_url='/redoc',          # ReDoc
        openapi_url='/docs.json',    # OpenAPI схема
        redirect_slashes=False
    )
    
    return app
```

После запуска приложения документация доступна по адресам:

- **Swagger UI:** `http://localhost:8080/docs`
- **ReDoc:** `http://localhost:8080/redoc`
- **OpenAPI схема (JSON):** `http://localhost:8080/docs.json`

### 2.3. Документирование эндпоинтов

#### Базовое документирование

```python
@router.post("/", response_model=UserResponse, status_code=201)
async def create_user(
    data: UserCreate,
    service: UserService = Depends(get_user_service)
):
    """
    Создание нового пользователя
    
    - **username**: уникальный логин пользователя (3-50 символов)
    - **email**: email адрес пользователя
    - **password**: пароль (минимум 8 символов)
    - **role**: роль пользователя (STUDENT, TEACHER, ADMIN)
    - **birth_date**: дата рождения
    
    Возвращает созданного пользователя с UUID
    """
    return await service.create_user(data)
```

#### Детальное документирование с примерами

```python
from fastapi import APIRouter, Body
from pydantic import Field

class UserCreate(BaseModel):
    username: str = Field(
        ..., 
        min_length=3, 
        max_length=50,
        description="Уникальный логин пользователя",
        example="ivan_ivanov"
    )
    email: EmailStr = Field(
        ...,
        description="Email адрес пользователя",
        example="ivan@example.com"
    )
    password: str = Field(
        ...,
        min_length=8,
        description="Пароль пользователя",
        example="SecurePass123"
    )
    role: UserRole = Field(
        ...,
        description="Роль пользователя в системе",
        example="STUDENT"
    )
    birth_date: date = Field(
        ...,
        description="Дата рождения",
        example="2005-01-15"
    )

@router.post(
    "/",
    response_model=UserResponse,
    status_code=201,
    summary="Создание пользователя",
    description="Создает нового пользователя в системе",
    responses={
        201: {
            "description": "Пользователь успешно создан",
            "content": {
                "application/json": {
                    "example": {
                        "id": "123e4567-e89b-12d3-a456-426614174000",
                        "username": "ivan_ivanov",
                        "email": "ivan@example.com",
                        "role": "STUDENT"
                    }
                }
            }
        },
        400: {"description": "Ошибка валидации данных"},
        409: {"description": "Пользователь с таким email/username уже существует"}
    },
    tags=["Users"]
)
async def create_user(
    data: UserCreate = Body(..., example={
        "username": "ivan_ivanov",
        "email": "ivan@example.com",
        "password": "SecurePass123",
        "role": "STUDENT",
        "birth_date": "2005-01-15",
        "first_name": "Иван",
        "surname": "Иванов"
    }),
    service: UserService = Depends(get_user_service)
):
    """
    ## Создание нового пользователя
    
    ### Требования:
    - Username должен быть уникальным
    - Email должен быть уникальным и валидным
    - Пароль минимум 8 символов
    - Роль: STUDENT, TEACHER или ADMIN
    
    ### Процесс:
    1. Валидация входных данных
    2. Проверка уникальности username и email
    3. Хеширование пароля
    4. Создание записи в БД
    5. Автоматическое создание профиля студента/преподавателя
    
    ### Возвращает:
    Данные созданного пользователя с присвоенным UUID
    """
    return await service.create_user(data)
```

### 2.5. Документирование аутентификации

```python
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials

security = HTTPBearer()

@router.get("/me")
async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security),
    service = Depends()
):
    """
    Получение данных текущего аутентифицированного пользователя
    
    Требуется JWT токен в заголовке Authorization: Bearer <token>
    """
    return await service.get_current_user(credentials.credentials)
```

### 3.3. Примеры запросов с документацией

#### Запрос: Login

```
POST {{base_url}}/api/users/auth
Content-Type: application/json

{
  "username": "ivan_ivanov",
  "password": "SecurePass123"
}
```

**Описание:**
```markdown
# Аутентификация пользователя

Выполняет вход пользователя в систему.

## Параметры
- `username` (string, required): логин пользователя
- `password` (string, required): пароль пользователя

## Ответ
- `200 OK`: успешная аутентификация, возвращает данные пользователя
- `401 Unauthorized`: неверные учетные данные

## Cookies
После успешной аутентификации устанавливаются cookies:
- `access_token`: JWT токен доступа (30 мин)
- `refresh_token`: токен обновления (7 дней)
```

**Tests Script:**
```javascript
// Сохранение токенов в переменные окружения
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

pm.test("Response has user data", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData).to.have.property('id');
    pm.expect(jsonData).to.have.property('username');
    pm.expect(jsonData).to.have.property('email');
});

// Извлечение токенов из cookies
var accessToken = pm.cookies.get('access_token');
var refreshToken = pm.cookies.get('refresh_token');

if (accessToken) {
    pm.environment.set("access_token", accessToken);
}
if (refreshToken) {
    pm.environment.set("refresh_token", refreshToken);
}

// Сохранение user_id
var jsonData = pm.response.json();
pm.environment.set("user_id", jsonData.id);
```

### 3.5. Документирование коллекции

**Добавление описания к коллекции:**

```markdown
# IT School API

RESTful API для управления платформой онлайн IT-школы.

## Базовый URL
- Development: `http://localhost:8080`
- Production: `https://api.itschool.example.com`

## Аутентификация
API использует JWT токены для аутентификации:
1. Выполните запрос `POST /api/users/auth` с логином и паролем
2. Получите `access_token` и `refresh_token` в cookies
3. Используйте `access_token` для авторизованных запросов
4. Обновляйте токен через `POST /api/users/refresh` при истечении

## Основные endpoints
- `/api/users` - управление пользователями
- `/api/courses` - управление курсами
- `/api/students` - управление студентами
- `/api/groups` - управление группами
- `/api/events` - управление событиями

## Пагинация
Все эндпоинты списков поддерживают пагинацию:
- `limit` - количество элементов (по умолчанию 10)
- `offset` - смещение (по умолчанию 0)

## Коды ответов
- `200 OK` - успешный запрос
- `201 Created` - ресурс создан
- `204 No Content` - успешное удаление
- `400 Bad Request` - ошибка валидации
- `401 Unauthorized` - требуется аутентификация
- `404 Not Found` - ресурс не найден
- `500 Internal Server Error` - ошибка сервера
```
